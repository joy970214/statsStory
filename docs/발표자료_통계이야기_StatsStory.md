## 1. 프로젝트 개요

### 🎯 프로젝트 시작 배경

**실무 니즈에서 출발한 프로젝트**

- 국토교통부 **통계누리 사이트**의 **"통계이야기"** 페이지
- **매달 카드뉴스 제작 업무** 담당
  - 예: 8월 통계이야기 → 7월에 올라온 통계 중 선택
  
**겪었던 문제점**

❌ 디자이너 배경이라 **통계 분석 경험 부족**
❌ 1달치 모든 통계를 **일일이 확인하기 불편**
❌ 어떤 통계를 선택할지 **방향 잡기 어려움**
❌ 통계표만 봐서는 **기획 포인트 파악 힘듦**

### ✨ 해결 아이디어

💡 **"AI 인사이트만 있어도 기획할 때 방향 잡기가 편하지 않을까?"**

- 최신 통계 **1달치만 자동 수집**
- AI가 각 통계의 **핵심 인사이트 자동 생성**
- 통계별 특징을 한눈에 비교
- **어떤 통계로 카드뉴스를 만들지 결정**

⚠️ **현실적 제약**
- 통계당 인사이트 생성: **5-10분 소요**
- CPU 환경의 한계로 실시간 분석은 어려움

### 🚀 향후 목표

- **HTML 차트 기반 시각화** 구현
- **카드뉴스 형태** 분석 결과 제공
- **실제 업무에 바로 활용** 가능한 수준으로 발전

## 2. 시스템 아키텍처

```
┌─────────────────────────┐
│   Frontend              │  React + TypeScript + Tailwind CSS
│   (사용자 UI)            │  • 통계 목록 보기
└───────────┬─────────────┘  • 실시간 진행률 확인
            │                 • 분석 결과 시각화
            │                 • AI 채팅 인터페이스 ⭐NEW
            │ REST API / SSE (실시간 통신)
            │
┌───────────▼─────────────┐
│   Backend               │  Python FastAPI
│   (API 서버)             │  • 통계 크롤링
└───────────┬─────────────┘  • 데이터 가공
            │                 • AI 분석 처리
            │                 • RAG 기반 질의응답 ⭐NEW
            │
    ┌───────┴────┬──────────┬──────────┬──────────┐
    ▼            ▼          ▼          ▼          ▼
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────────┐
│ Crawler │ │ Ollama  │ │  Cache  │ │  Data   │ │ VectorDB │
│ Service │ │ Service │ │ Storage │ │ Storage │ │(ChromaDB)│
│         │ │         │ │         │ │         │ │          │
│Selenium │ │Llama3.1 │ │ JSON    │ │ JSON    │ │벡터검색  │
│크롤링    │ │AI분석   │ │캐싱     │ │저장     │ │RAG지원   │⭐NEW
└─────────┘ └─────────┘ └─────────┘ └─────────┘ └──────────┘
            ▲                                     ▲
            │                                     │
            └──────────── 채팅 질의응답 ───────────┘
                        (RAG 방식)
```

**주요 서비스 역할:**
- **Crawler Service**: Selenium 기반 웹 크롤링 (메타데이터 + 데이터 수집)
- **Ollama Service**: Llama 3.1 로컬 LLM으로 AI 인사이트 생성 + 채팅 응답
- **Cache Storage**: 수집 결과 캐싱 (metadata, collected_data, analysis_results)
- **Data Storage**: 통계 데이터 JSON 저장 및 관리
- **VectorDB (ChromaDB)**: 통계 데이터 벡터화 저장, RAG 기반 의미 검색 ⭐NEW

**※ MCP Server**: 코드에만 존재, 실제 사용 안 함 (향후 확장용)

## 5. 주요 기능 - AI 채팅 (RAG)

### RAG 기반 대화형 분석 ⭐NEW

**위치**: 분석 결과 페이지 내 아코디언 UI

**작동 방식:**
1. 사용자가 통계 데이터에 대해 질문
2. ChromaDB에서 질문과 **관련된 데이터만 검색** (상위 8개)
3. AI 인사이트 + 검색된 데이터 → Ollama에게 전달
4. 컨텍스트 기반 정확한 답변 생성

**주요 특징:**
✅ **RAG (Retrieval-Augmented Generation)** 방식
   - 전체 데이터가 아닌 관련 데이터만 검색
   - 토큰 제한 회피 + 정확도 향상

✅ **대화 히스토리 유지**
   - 이전 대화 맥락 유지
   - 연속된 질문 가능

✅ **실시간 응답**
   - 로컬 LLM으로 빠른 응답
   - 데이터 외부 유출 없음

**사용 예시:**
- "2023년 서울 주택 미분양 현황은?"
- "전년 대비 증가율이 가장 높은 지역은?"
- "최근 3년간 추이를 요약해줘"

## 5. 주요 기능 - 실시간 진행률 표시

### SSE (Server-Sent Events) 기술

📡 **서버 → 클라이언트 실시간 데이터 푸시**

### 표시되는 정보

1. **현재 진행 단계** (메타데이터 수집, 테이블 수집 등)
2. **진행률 (%)** - 프로그레스바로 시각화
3. **현재 작업 내용** - 상세 메시지 표시
4. **예상 소요 시간** - 사용자 대기 시간 안내
5. **취소 기능** - 언제든 작업 중단 가능

### 장점

✨ 작업이 멈춘 것처럼 보이지 않음
✨ 투명한 프로세스 제공
✨ 사용자 이탈 방지

## 6. 핵심 기술 - AI 분석 엔진 (Ollama)

### Ollama 로컬 LLM 활용

🧠 **오픈소스 로컬 AI로 데이터 프라이버시 보장**

### 기술 스펙 & 제약사항

- **모델**: Llama 3.1 (8B Instruct)
- **실행 환경**: 로컬 서버 (CPU 전용, GPU 없음)
- **장점**: 
  - 비용 없음 (API 호출 비용 0원)
  - 데이터 외부 유출 없음
- **제약사항**:
  - 16K 토큰 제한
  - CPU 성능 제한
  - Claude/GPT 대비 분석 품질 차이
- **극복 방법**:
  - 통계표별 분할 분석으로 토큰 제한 회피
  - CPU 최적화 모델 사용

### 분석 프로세스

1. **데이터 구조 분석** - 컬럼, 연도, 단위 파악
2. **기본 통계량 계산** - 평균, 중앙값, 최대/최소값
3. **시계열 트렌드 분석** - 증가/감소 패턴 파악
4. **주요 변화 시점 탐지** - 급격한 변화 지점 식별
5. **통계표별 개별 인사이트** - 각 표별 특징 분석
6. **종합 인사이트** - 전체 통계의 의미 해석

### 결과 & 현실

💡 **전문가가 아니어도 쉽게 이해할 수 있는 텍스트 인사이트**
🔒 **데이터가 외부로 유출되지 않는 안전한 분석**
📊 **구조화된 데이터로 추후 시각화 가능**

⚠️ **성능 한계**
- 통계표당 3~10분 소요 (CPU 환경)
- 대형 통계는 더 오래 걸림
- 실시간 분석보다는 **배치 처리에 적합**

💬 **품질 비교 (체감)**
- Claude/GPT 대비 30-40% 품질 차이
- 기본 통계는 잘 뽑음, 복잡한 해석은 약함
- 하지만 업무 목적(통계 선정)으로는 충분

🔧 **모니터링**
- SSE로 실시간 진행률 표시
- FastAPI 로그, 브라우저 콘솔
- 별도 모니터링 도구 없음 (개인 프로젝트)

## 8. 개발 방법론 - 바이브 코딩

### 개발 규모

- **개발 기간**: 2-3주 (40-60시간)
- **개발 인원**: 1명 (디자이너 배경)
- **작업 시간**: 주말 + 퇴근 후 2-3시간
- **산출물**: 일자별 개발 일지 20여 개 (docs 폴더)

### 바이브 코딩이란?

**AI와 대화하며 코드를 작성하는 방식**
- Claude (Cursor AI)와 실시간 대화
- 문제 설명 → AI가 코드 제안 → 테스트 → 개선
- 전통적인 검색 & 복붙이 아닌 **협업 개발**

### 바이브 코딩의 장점

✅ **비개발자도 복잡한 시스템 구축 가능**
   - 디자이너가 풀스택 개발 완성
   - 전통적 방식 대비 2-3배 빠름
   
✅ **프론트엔드 개발 프로세스**
   - 와이어프레임 손그림 → Claude에게 설명
   - React 컴포넌트 제안 → 테스트 → 개선
   - Tailwind CSS로 스타일링
   
✅ **디버깅 전략**
   - 에러 메시지 Claude에게 복붙
   - console.log, print 문 도배
   - Chrome DevTools, FastAPI /docs 활용
   - 문제를 작게 나누어 해결

### 한계점

❌ AI가 제안한 코드를 **이해 없이 사용하면 위험**
❌ 복잡한 버그는 **직접 디버깅 필요** (비동기, SSE, CORS 등)
❌ 프로젝트 전체 구조 파악은 **개발자 몫**
❌ **주석과 문서 부족** - 향후 개선 필요

## 15. Q&A

### 자주 묻는 질문

**Q1. 왜 이 프로젝트를 만들게 되었나요?**
A. 매달 "통계이야기" 카드뉴스를 제작해야 하는데, 1달치 통계를 일일이 확인하고 분석하는 게 너무 불편했습니다. 디자이너 배경이라 통계 분석 경험도 없어서, AI가 인사이트만 뽑아줘도 기획 방향을 잡기 훨씬 수월할 것 같아 시작했습니다.

**Q2. 실제로 업무에 활용하고 있나요?**
A. 현재 1단계(데이터 수집 + 인사이트)까지 완료했습니다. 다만 통계당 5-10분 걸려서 실시간 활용은 어렵고, 전날 밤 배치로 돌려놓고 아침에 결과를 확인하는 방식으로 활용 중입니다. 성능 개선이 필요한 상태입니다.

**Q3. 크롤링이 불법이 아닌가요?**
A. 공공데이터는 개방 원칙에 따라 활용 가능합니다. 서버에 부담을 주지 않도록 적절한 대기 시간을 두고 있습니다.

**Q4. AI 분석 비용은 얼마나 드나요?**
A. Ollama 로컬 LLM을 사용하여 API 비용이 전혀 들지 않습니다. GPU 없이 CPU만으로도 동작하지만, 통계표당 3-10분 소요되어 성능이 느린 것이 단점입니다. CPU 노트북에서도 동작하는 완전 로컬 솔루션으로 만들었고, 품질 차이는 있지만 업무에는 충분히 활용 가능한 수준입니다.

**Q5. 개발 경험이 없는데 어떻게 만들었나요?**
A. 바이브 코딩으로 개발했습니다. Claude AI(Cursor)와 대화하며 "이런 기능을 만들고 싶다"고 설명하면 AI가 코드를 제안하고, 테스트하면서 개선해나갔습니다. 물론 완전 자동은 아니고 AI 제안을 이해하고 디버깅하는 과정이 필요했습니다.

**Q6. AI 채팅 기능은 어떻게 작동하나요?**
A. RAG (Retrieval-Augmented Generation) 방식입니다. 질문이 들어오면 ChromaDB에서 벡터 유사도로 관련 데이터만 검색(상위 8개)하고, AI 인사이트와 함께 Ollama에게 전달합니다. 전체 데이터가 아닌 필요한 부분만 사용해서 토큰 제한도 회피하고 정확도도 높아집니다.
