## 개요

기존 URL 기반 캐시 시스템의 한계를 극복하기 위해 통계명 기반 캐시 조회 시스템으로 전면 개선하고, IBSheet 원본 테이블 구조를 완벽하게 재현하는 시스템을 구현했습니다. 또한 다수의 새로운 통계 데이터를 수집하여 캐시 풀을 확장했습니다.

## 주요 변경사항

### 1. 통계명 기반 캐시 시스템 구현

#### 문제점
- 기존 URL 기반 캐시 매칭이 불안정하여 캐시 히트율 저조
- 7개의 기존 캐시 파일이 있음에도 불구하고 재수집 발생
- 한국어 통계명 정규화 처리 부족으로 정확한 매칭 실패

#### 해결방법

**새로운 캐시 조회 시스템 구현**
```python
# backend/app/services/data_storage.py
def get_cached_data_by_name(self, stat_name: str, max_age_hours: int = 24) -> Tuple[Optional[StatMetadata], Optional[List[StatData]]]:
    """통계명 기반으로 캐시된 데이터 검색"""

    metadata_pattern = os.path.join(self.metadata_dir, "*_metadata.json")
    metadata_files = glob.glob(metadata_pattern)

    for metadata_file in metadata_files:
        with open(metadata_file, 'r', encoding='utf-8') as f:
            data = json.load(f)

        metadata_title = data.get('metadata', {}).get('title', '')

        # 통계명 매칭 (정규화된 비교)
        if self._normalize_stat_name(metadata_title) == self._normalize_stat_name(stat_name):
            # 캐시 히트 - 메타데이터 및 통계 데이터 반환
            return metadata_obj, stat_data_list

    return None, None

def _normalize_stat_name(self, name: str) -> str:
    """한국어 통계명 정규화"""
    # 괄호 안의 영문 제거, 공백/특수문자 정리
    normalized = re.sub(r'\([^)]*\)', '', name)
    normalized = re.sub(r'[^\w가-힣]', '', normalized)
    return normalized.lower().strip().replace(' ', '')
```

### 2. IBSheet 완전 재구성 시스템 구현

#### 문제점
- 기존 IBSheet 재구성이 단순화된 테이블만 생성
- 원본 IBSheet의 복잡한 헤더 구조와 병합 셀 무시
- 사용자 요구사항 "수집 당시 모양 그대로" 미구현

#### 해결방법

**IBSheet 원본 구조 완전 재현 시스템**
```typescript
// frontend/src/components/EnhancedBasicStatisticsViewer.tsx
const reconstructIBSheetTable = (rawData: any[]) => {
  rawData.forEach((item: any, tableIndex: number) => {
    if (item?.data?._table_data && item?.data?._table_structure) {
      const tableData = JSON.parse(item.data._table_data);
      const tableStructure = JSON.parse(item.data._table_structure);

      // 1. IBSheet 원본 크기 그리드 초기화
      const maxRow = Math.max(...tableData.map((row: any) => row.row_index)) + 1;
      const maxCol = tableStructure.cols || 84;

      // 2. 실제 row_index, col_index 기반 셀 배치
      tableData.forEach((rowInfo: any) => {
        const sortedCells = rowInfo.cells.sort((a: any, b: any) => a.col_index - b.col_index);

        sortedCells.forEach((cell: any, cellIndex: number) => {
          // 3. colspan 자동 계산 (다음 셀까지의 거리)
          let colspan = 1;
          if (cellIndex < sortedCells.length - 1) {
            const nextColIndex = sortedCells[cellIndex + 1].col_index;
            colspan = nextColIndex - colIndex;
          }

          // 4. 셀 데이터 완전 복원
          grid[rowIndex][colIndex] = {
            value: cell.value?.value || '',
            colspan: colspan,
            isHeader: cell.is_header || rowInfo.is_header,
            // ... 기타 원본 속성들
          };
        });
      });
    }
  });
};
```

3. 셀 간 거리 계산으로 colspan 자동 결정
4. 병합 셀, 헤더 계층, 스타일 정보 완전 보존

### 통계명 정규화 알고리즘
```python
def _normalize_stat_name(self, name: str) -> str:
    # 1. 괄호 안의 영문 제거: "미분양주택현황보고(Unsold New Housings)" → "미분양주택현황보고"
    normalized = re.sub(r'\([^)]*\)', '', name)
    # 2. 특수문자 제거: "공동주택 현황" → "공동주택현황"
    normalized = re.sub(r'[^\w가-힣]', '', normalized)
    # 3. 대소문자 통일 및 공백 제거
    return normalized.lower().strip().replace(' ', '')
```
