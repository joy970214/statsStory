## 3. 프론트엔드 `API_ORIGIN` 도입

### 문제
- 개발 환경에서는 `package.json`의 `proxy` 설정(`/api` 상대경로)으로 백엔드 호출이 가능했지만
- **프로덕션 빌드(외부 배포)에서는 proxy가 동작하지 않아** `/api` 호출이 프론트엔드 호스트로 향함 → 404
- 또한 SSE는 CRA dev proxy의 buffering 이슈로 dev에서도 백엔드 직결이 필요

### 해결 — `frontend/src/services/api.ts`
```ts
export const API_ORIGIN = process.env.NODE_ENV === 'production'
  ? `${window.location.protocol}//${window.location.hostname}:17202`
  : '';
const API_BASE_URL = `${API_ORIGIN}/api`;
const SSE_ORIGIN = API_ORIGIN || `${window.location.protocol}//${window.location.hostname}:17202`;
```

- **개발**: `API_ORIGIN = ''` → 기존 proxy 경유 유지
- **프로덕션**: 현재 접속 호스트 + `:17202` 로 직접 호출
- **SSE**: 항상 백엔드 직결

### 일괄 갱신된 fetch 경로
- `App.tsx`, `CollectedStatsViewer.tsx`, `EnhancedBasicStatisticsViewer.tsx`
- `StatDetailViewer.tsx`, `StatDistributionViewer.tsx`, `StatSummaryViewer.tsx`
- `TableAnalysisViewer.tsx`
- 패턴: `fetch('/api/...')` → `fetch(\`${API_ORIGIN}/api/...\`)`

## 4. Linux 환경에서 크롤러 실행 지원

### 문제
- 기존 코드는 `webdriver-manager`가 OS/아키텍처를 감지해 ChromeDriver를 받아오는 구조였으나
  - Linux 서버에 시스템 Chrome이 없는 경우 실패
  - 네트워크가 제한된 환경에서 매번 드라이버 다운로드 시도

### 해결 — `backend/app/crawlers/base_crawler.py`
```python
def _resolve_chrome_binary() -> str | None:
    """CHROME_BINARY 환경변수 → 사용자 영역 추출 경로 → None(기본 탐색) 순으로 해석"""
    env_path = os.environ.get("CHROME_BINARY")
    if env_path and Path(env_path).exists():
        return env_path
    local_path = Path.home() / "chrome" / "opt" / "google" / "chrome" / "google-chrome"
    if local_path.exists():
        return str(local_path)
    return None
```

- **Selenium Manager(셀레늄 4.6+ 내장)** 가 설치된 Chrome 버전에 맞는 ChromeDriver를 자동 해석
- `webdriver-manager` 의존 제거
- `CHROME_BINARY` 환경변수 또는 `~/chrome/opt/google/chrome/google-chrome` 경로로 사용자 영역에 추출한 Chrome 실행 가능 (root 권한 불필요)

### 변경 파일
- `backend/app/crawlers/base_crawler.py` — 헬퍼 함수 신규 + 적용
- `backend/app/crawlers/legacy/molit_crawler.py` — 동일 패턴 적용
- `backend/app/crawlers/legacy/optimized_molit_crawler.py` — 동일 패턴 적용

## 6. 배포 환경 정리

### 현재 가동 상태 (Linux 서버)
- 백엔드: `python main.py` → `0.0.0.0:17202`
- 프론트엔드: `PORT=17200 HOST=0.0.0.0 npm start` → `0.0.0.0:17200`
- 외부 접속: `http://175.213.188.161:17200`

### 실행 예시 (Linux)
```bash
# 백엔드
cd backend && .venv/bin/python main.py

# 프론트엔드
cd frontend && PORT=17200 HOST=0.0.0.0 npm start

# (선택) Chrome 바이너리 경로 지정
export CHROME_BINARY=/path/to/google-chrome
```

> `start.bat` 은 Windows 개발 환경용으로 유지. Linux 서버용 스크립트 별도 정비는 후속 작업.
