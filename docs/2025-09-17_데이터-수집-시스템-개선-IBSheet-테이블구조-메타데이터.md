# 2025-09-17 데이터 수집 시스템 개선: IBSheet 테이블구조 및 메타데이터

## 📋 개요

국토교통 통계누리 크롤링 시스템에서 발견된 데이터 수집 품질 문제를 해결하고, 사용자 경험을 개선하기 위한 시스템 개선 작업을 진행했습니다.

## 🔍 발견된 문제점들

### 1. IBSheet 데이터 구조 문제
- **문제**: `ibsheet_cell_0`, `ibsheet_cell_1` 형태의 순차적 인덱스만 저장
- **결과**: 원본 테이블의 행/열 구조 정보 손실
- **영향**: 사용자가 데이터 이해하기 어려움

### 2. 메타데이터 수집 불완전
- **문제**: 통계정보 탭과 관련용어 탭에서 데이터 수집이 제대로 되지 않음
- **누락**: 검색분야, 담당부서, 상세 통계정보 등 중요 메타데이터
- **영향**: 통계 이해도 저하

### 3. 원본 테이블 형태 재현 불가
- **문제**: 프론트엔드에서 추정으로만 테이블 구조 복원
- **결과**: 실제 원본 웹사이트와 다른 레이아웃
- **영향**: 데이터 신뢰성 문제

## ✅ 해결방법 및 구현 내용

### 1. IBSheet 데이터 수집 방식 개선

#### 변경 전 (문제점)
```python
# 순차적 셀 인덱스만 저장
for i, cell in enumerate(cells):
    key = f"ibsheet_cell_{i}"
    ibsheet_data[key] = cell_value
```

#### 변경 후 (해결책)
```python
# 구조화된 테이블 정보 포함
ibsheet_data['_table_structure'] = {
    'type': 'IBSheet',
    'rows': row_count,
    'cols': col_count,
    'sheet_name': sheet_name
}

ibsheet_data['_table_headers'] = headers
ibsheet_data['_table_data'] = [
    {
        'row_index': row_idx,
        'is_header': row_idx == 0,
        'cells': [
            {
                'col_index': col_idx,
                'col_name': col_name,
                'value': cell_value,
                'is_header': is_header
            }
        ]
    }
]
```

**개선 효과**:
- ✅ 원본 테이블 구조 완전 보존
- ✅ 헤더와 데이터 구분 명확
- ✅ 컬럼명 정보 유지
- ✅ 행/열 위치 정보 보존

### 2. 메타데이터 수집 로직 강화

#### 기본 정보 수집 추가
```python
# 검색분야, 담당부서 추가 수집
metadata_info.update({
    'search_field': '',  # 검색분야
    'responsible_department': '',  # 담당부서
    'statistical_info': {},  # 통계정보 상세
    'major_items': {},  # 주요항목
    'meaning_analysis': {},  # 의미분석
    'terminology': {}  # 관련용어
})
```

#### 통계정보 탭 개선
```python
# table caption이 "통계정보"인 테이블 우선 처리
stat_info_tables = driver.find_elements(
    By.XPATH,
    "//table[caption[contains(text(), '통계정보')] or caption[contains(text(), '통계 정보')]]"
)

# tbody 내 th:td 매핑으로 정확한 수집
for row in tbody.find_elements(By.TAG_NAME, "tr"):
    th_elements = row.find_elements(By.TAG_NAME, "th")
    td_elements = row.find_elements(By.TAG_NAME, "td")

    if th_elements and td_elements:
        key = th_elements[0].text.strip()
        value = td_elements[0].text.strip()
        metadata_info['statistical_info'][key] = value
```

#### 관련용어 탭 개선
```python
# 3개 섹션 분리 수집
sections = [
    {'name': 'major_items', 'keywords': ['주요항목', '주요 항목']},
    {'name': 'meaning_analysis', 'keywords': ['의미분석', '의미 분석']},
    {'name': 'terminology', 'keywords': ['관련용어', '관련 용어']}
]

# 테이블 제목 기반 섹션 구분
for section in sections:
    if any(keyword in table_title for keyword in section['keywords']):
        # 해당 섹션에 키-값 쌍 저장
        section['target'][key] = value
```

**개선 효과**:
- ✅ 상세 통계정보 완전 수집
- ✅ 구조화된 관련용어 분류
- ✅ 검색분야, 담당부서 정보 추가
- ✅ 기존 호환성 유지

### 3. 프론트엔드 원본 테이블 표시 개선

#### 기존 문제
```tsx
// 추정에 의존한 테이블 재구성
const estimatedCols = Math.ceil(Math.sqrt(cellData.length)) || 5;
```

#### 개선된 방식
```tsx
// 백엔드에서 제공하는 구조화된 데이터 활용
if (data._table_structure && data._table_data) {
    const tableStructure = data._table_structure;
    const tableData = data._table_data;

    // 정확한 행/열 구조로 테이블 재구성
    tableData.forEach((rowInfo) => {
        rowInfo.cells.forEach((cellInfo) => {
            row[cellInfo.col_index] = {
                value: cellInfo.value.value,
                type: cellInfo.value.unit,
                isHeader: rowInfo.is_header,
                colName: cellInfo.col_name
            };
        });
    });
}
```

**개선 효과**:
- ✅ 실제 원본 웹사이트와 동일한 테이블 구조
- ✅ 정확한 헤더 인식 및 강조
- ✅ 컬럼명 표시 및 툴팁 제공
- ✅ 기존 방식과 호환성 유지
