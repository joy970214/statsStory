## 1. 프로젝트 개요

### 왜 만들게 되었나?

**배경: 매달 "통계이야기" 카드뉴스 제작 업무**

**문제점**
- 디자이너 배경 → 통계 분석 경험 부족
- 1달치 통계를 일일이 확인하기 불편
- 어떤 통계로 카드뉴스를 만들지 방향 잡기 어려움

**해결 아이디어**
> "AI 인사이트만 있어도 기획할 때 방향 잡기가 편하지 않을까?"

**솔루션**
- 최신 통계 1달치 **자동 수집**
- AI가 각 통계의 **핵심 인사이트 자동 생성**
- 통계별 특징을 한눈에 비교 → **빠른 의사결정**

## 1. 프로젝트 개요 (계속)

**현재 (1단계 완료)**
- 통계 자동 수집

## 2. 시스템 아키텍처

```
┌──────────────────────┐
│   Frontend (React)   │  사용자 UI
└──────────┬───────────┘  - 통계 목록/선택
           │              - 실시간 진행률
           │              - AI 채팅
           │ REST API / SSE
┌──────────▼───────────┐
│  Backend (FastAPI)   │  API 서버
└──────────┬───────────┘  - 크롤링/분석
           │
    ┌──────┴─────┬────────┬────────┐
    ▼            ▼        ▼        ▼
┌────────┐  ┌────────┐ ┌──────┐ ┌─────────┐
│Selenium│  │Ollama  │ │Cache │ │ChromaDB │
│크롤링   │  │AI분석  │ │저장  │ │RAG검색  │
```

**핵심 컴포넌트**
- **Selenium**: 동적 웹 크롤링

- **ChromaDB**: 벡터 검색 (RAG)

## 3. 핵심 기능

### ① 통계 자동 수집
- 최근 1개월 통계 크롤링
- 엑셀 다운로드 + 직접 수집 (이중 전략)
- 실시간 진행률 표시 (SSE)

### ② AI 인사이트 생성
- 통계표별 자동 분석
- 트렌드, 변화 시점 탐지
- 쉬운 텍스트 요약

### ③ AI 채팅 (RAG)
- 통계 데이터에 대해 질문
- 벡터 검색으로 관련 데이터만 추출
- 정확한 답변 생성

## 4. 핵심 기술 ① - 동적 웹 크롤링

### 도전 과제
- JavaScript 기반 동적 사이트
- 복잡한 테이블 구조 (병합 셀)
- 다양한 데이터 형식

### 해결 방법: 이중 전략

**1순위: 엑셀 다운로드**
- Selenium으로 다운로드 버튼 클릭
- Pandas로 파일 파싱
- 가장 정확, 구조화 쉬움

**2순위: 직접 테이블 수집**
- Selenium + BeautifulSoup
- JavaScript 로딩 후 DOM 파싱
- 백업 전략

---

## 4. 핵심 기술 ② - AI 분석 (Ollama)

### Ollama 로컬 LLM

**모델**: Llama 3.1 (8B Instruct)
**실행 환경**: CPU 전용 (GPU 없음)

**장점**
- 비용 0원 (API 호출 없음)
- 데이터 외부 유출 없음
- 완전 로컬 솔루션

**제약사항 & 극복**
- 16K 토큰 제한 → **통계표별 분할 분석**
- CPU 성능 제한 → **배치 처리** 활용
- 통계당 5-10분 소요 → 밤사이 자동 처리

---

## 4. 핵심 기술 ③ - RAG 채팅

### RAG (Retrieval-Augmented Generation)

**작동 방식**
1. 사용자 질문 입력
2. ChromaDB에서 **관련 데이터만 벡터 검색** (상위 8개)
3. AI 인사이트 + 검색 데이터 → Ollama
4. 컨텍스트 기반 정확한 답변 생성

**장점**
- 토큰 제한 회피 (전체 데이터 X)
- 정확도 향상 (관련 데이터만 사용)
- 대화 맥락 유지

**예시 질문**
- "2023년 서울 미분양 현황은?"
- "전년 대비 증가율 높은 지역은?"

## 5. 개발 방법론 - 바이브 코딩

### 바이브 코딩이란?

**AI(Claude)와 대화하며 코드 작성**
- 문제 설명 → AI 제안 → 테스트 → 개선
- 검색 & 복붙이 아닌 **협업 개발**

**개발 규모**
- 기간: 2-3주 (40-60시간)
- 인원: 1명 (디자이너 배경)
- 산출물: 일자별 개발 일지 20여 개

**결과**
- 비개발자가 **풀스택 시스템** 구축
- 전통 방식 대비 **2-3배 빠름**
- AI 코드 이해 없이 사용하면 위험
- 복잡한 버그는 직접 디버깅 필요

## 6. 주요 챌린지 & 해결책

### ① AI 분석 - 토큰 제한 문제

**문제**: 통계 전체 데이터를 한 번에 분석 → 16K 토큰 초과

**해결**: **통계표별 분할 분석**
- 통계 전체 → 개별 통계표로 분할
- 각 표별 인사이트 생성 후 종합
- 프롬프트 최적화

### ② 동적 테이블 크롤링

**문제**: IBSheet JavaScript 라이브러리 렌더링

**해결**: Selenium 완전 로딩 대기 + DOM 파싱

### ③ 실시간 진행률 표시

**문제**: 긴 작업 시간 (1-2분) → 사용자 이탈

**해결**: SSE (Server-Sent Events) 도입
