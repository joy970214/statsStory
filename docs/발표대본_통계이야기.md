# 통계이야기 발표 대본

## 슬라이드 1: 타이틀

안녕하세요. 오늘 제가 소개해드릴 프로젝트는 '통계누리 최신 통계 분석' 시스템입니다.
통계이야기 생성을 위한 AI 기반 통계 분석 시스템인데요, 지금부터 자세히 설명드리겠습니다.

---

## 슬라이드 2: 목차

오늘 발표는 크게 8가지 순서로 진행하겠습니다.
먼저 프로젝트를 만들게 된 배경과 시스템 구조를 설명드리고,
핵심 기능과 사용된 기술들을 소개한 다음,
개발 과정에서 마주한 문제와 해결 방법들을 공유드리겠습니다.
마지막으로 현재 한계점과 향후 계획을 말씀드린 뒤, 시연을 보여드리겠습니다.

---

## 슬라이드 3: 프로젝트 개요 - 배경

먼저 이 프로젝트를 왜 만들게 되었는지 말씀드리겠습니다.

제가 맡고 있는 업무 중 하나가 매달 "통계이야기"라는 카드뉴스를 제작하는 건데요,
저는 디자이너 배경이라 통계 분석 경험이 부족합니다.
그래서 1달치 통계를 일일이 확인하면서 어떤 내용으로 카드뉴스를 만들지 방향을 잡는 게 
항상 어려웠습니다.

그러다 이런 생각이 들었습니다.
"AI가 각 통계의 인사이트만 뽑아줘도 기획할 때 방향 잡기가 훨씬 편하지 않을까?"

그래서 만든 솔루션이 바로 이겁니다.
최신 통계 1달치를 자동으로 수집하고,
AI가 각 통계의 핵심 인사이트를 자동으로 생성해주고,
통계별 특징을 한눈에 비교해서 빠르게 의사결정을 할 수 있도록 했습니다.

---

## 슬라이드 4: 프로젝트 개요 - 현재 상태

현재 1단계가 완료된 상태입니다.
통계 자동 수집 기능이 있고, AI 인사이트 생성, 그리고 RAG 기반 AI 채팅 기능이 구현되어 있습니다.
다만 CPU 환경에서 작동하기 때문에 통계당 5분에서 10분 정도 시간이 걸립니다.

향후 2단계로는 HTML 차트 기반 시각화를 추가하고,
카드뉴스 형태로 결과를 제공하며,
성능 최적화와 인사이트 품질을 개선할 계획입니다.

---

## 슬라이드 5: 시스템 아키텍처

시스템 구조를 간단히 설명드리겠습니다.

가장 왼쪽에 React(리액트)로 만든 프론트엔드가 있고,
REST API와 SSE(Server-Sent Events, 서버 센트 이벤츠)를 통해 
FastAPI(패스트 에이피아이) 백엔드와 통신합니다.

백엔드에서는 4개의 핵심 컴포넌트를 사용합니다.
Selenium(셀레니움)으로 웹 크롤링을 하고,
Ollama(올라마)로 로컬 LLM(엘엘엠)을 돌려서 AI 분석을 하고,
Cache(캐시)에 결과를 저장하며,
ChromaDB(크로마디비)로 벡터 검색을 수행합니다.

---

## 슬라이드 6: 핵심 기능

핵심 기능은 크게 3가지입니다.

첫 번째, 통계 자동 수집입니다.
최근 1개월 통계를 크롤링하는데, 엑셀 다운로드와 직접 수집이라는 이중 전략을 사용합니다.
그리고 실시간 진행률을 SSE로 표시합니다.

두 번째, AI 인사이트 생성입니다.
통계표별로 자동 분석을 하고, 트렌드와 변화 시점을 탐지하며, 쉬운 텍스트로 요약해줍니다.

세 번째, AI 채팅 기능인데요, RAG(래그) 방식을 사용합니다.
통계 데이터에 대해 질문하면, 벡터 검색으로 관련 데이터만 추출해서 정확한 답변을 생성합니다.

## 슬라이드 12: 핵심 기술 ① - 동적 웹 크롤링

첫 번째 핵심 기술은 동적 웹 크롤링입니다.

통계누리 사이트는 JavaScript(자바스크립트) 기반의 동적 사이트라서 
일반적인 크롤링으로는 데이터를 가져올 수 없었습니다.
게다가 테이블 구조가 복잡하고 데이터 형식도 다양했습니다.

그래서 이중 전략을 사용했습니다.
1순위로는 Selenium(셀레니움)으로 엑셀 다운로드 버튼을 클릭해서 
Pandas(판다스)로 파일을 파싱하는 방법을 사용합니다. 이게 가장 정확하고 구조화하기 쉽습니다.
2순위는 백업 전략으로, Selenium과 BeautifulSoup(뷰티풀숲)을 사용해서 
JavaScript 로딩 후 DOM을 직접 파싱하는 방법입니다.

---

## 슬라이드 13: 핵심 기술 ② - AI 분석 (Ollama)

두 번째는 AI 분석인데요, Ollama(올라마)를 사용했습니다.

Llama 3.1(라마 쓰리 포인트 원) 8B Instruct(에잇 비 인스트럭트) 모델을 CPU 환경에서 돌립니다.

장점은 확실합니다.
API 호출이 없어서 비용이 0원이고,
데이터가 외부로 유출되지 않으며,
완전히 로컬에서 작동하는 솔루션입니다.

하지만 제약사항도 있습니다.
16K 토큰 제한이 있어서 통계표별로 분할 분석을 해야 했고,
CPU 성능 제한과 통계당 5-10분이 소요되는 점은 아직 개선이 필요합니다.

---

## 슬라이드 14: 핵심 기술 ③ - RAG 채팅

세 번째는 RAG(래그), 즉 Retrieval-Augmented Generation(리트리벌 오그멘티드 제너레이션)입니다.

작동 방식은 이렇습니다.
사용자가 질문을 입력하면,
ChromaDB(크로마디비)에서 관련 데이터만 벡터 검색으로 상위 8개를 찾아냅니다.
그 다음 AI 인사이트와 검색된 데이터를 Ollama(올라마)에 넣어서
컨텍스트 기반의 정확한 답변을 생성합니다.

이 방식의 장점은, 
전체 데이터를 다 넣지 않아서 토큰 제한을 회피할 수 있고,
관련 데이터만 사용하니까 정확도가 향상되며,
대화 맥락도 유지할 수 있다는 겁니다.

예를 들어 "2023년 서울 미분양 현황은?"이나 
"전년 대비 증가율 높은 지역은?" 같은 질문에 답변할 수 있습니다.

---

## 슬라이드 15: 개발 방법론 - 바이브 코딩

이 프로젝트는 바이브 코딩, 즉 AI와 대화하며 코드를 작성하는 방식으로 개발했습니다.

Claude(클로드)와 문제를 설명하고, AI가 제안하고, 테스트하고, 개선하는 과정을 반복했습니다.

개발 규모를 보면,
3-4주 정도, 약 60시간 투입했고,
1명이서 풀스택 시스템을 구축했으며,
일자별 개발 일지가 20개 이상 나왔습니다.

결과적으로 전통 방식 대비 2-3배 빠르게 개발할 수 있었습니다.
다만 주의할 점은, AI 코드를 이해하지 않고 사용하면 위험하고,
복잡한 버그는 직접 디버깅이 필요하다는 겁니다.

---

## 슬라이드 16: 주요 챌린지 & 해결책

개발 과정에서 3가지 큰 문제가 있었습니다.

첫 번째, AI 분석의 토큰 제한 문제입니다.
통계 전체 데이터를 한 번에 분석하려니 16K 토큰을 초과했습니다.
이건 통계표별로 분할 분석하는 방식으로 해결했습니다.

두 번째, 동적 테이블 크롤링 문제입니다.
IBSheet(아이비시트) JavaScript 라이브러리로 렌더링되는 테이블이었는데,
Selenium으로 완전히 로딩될 때까지 대기한 후 DOM을 파싱하는 방식으로 해결했습니다.

세 번째, 실시간 진행률 표시 문제입니다.
작업 시간이 1-2분 정도 걸리니까 사용자가 이탈할 수 있어서,
SSE(Server-Sent Events, 서버 센트 이벤츠)를 도입해서 실시간 진행률을 보여줬습니다.

---

## 슬라이드 17: 한계점 & 향후 계획

현재 한계점을 솔직히 말씀드리겠습니다.

가장 큰 이슈는 성능 문제입니다.
통계표당 3-10분이 소요되고, 대형 통계는 1시간 이상 걸려서 실시간 사용이 불가능합니다.

두 번째는 분석 품질입니다.
Claude(클로드)나 GPT(지피티) 대비 50-60% 정도 품질 차이가 있고,
복잡한 해석에는 한계가 있습니다.

향후 개선 방향은,
인사이트 품질을 개선하고,
Chart.js(차트 제이에스)로 데이터 시각화를 추가하며,
카드뉴스 형태로 구현하는 것입니다.

---

## 슬라이드 18: 기타 주요 기능

소개하지 못한 자잘한 기능들도 있습니다.

캐시 시스템으로 분석 결과를 자동 저장해서 중복 작업을 방지하고,
통계표 삭제 기능으로 원본 파일과 벡터 DB를 완전히 삭제할 수 있으며,
인사이트 문서를 MD(마크다운), PDF(피디에프) 형태로 다운로드할 수 있습니다.

또한 SSE 기반으로 실시간 작업 진행률을 추적하고,
새로운 통계를 자동으로 인식해서 메타데이터를 생성하며,
진행 중인 작업을 안전하게 취소할 수도 있습니다.

---

## 슬라이드 19: 기술 스택

마지막으로 기술 스택을 정리하면,

Frontend(프론트엔드)는 React 19(리액트 19), TypeScript(타입스크립트), Tailwind CSS(테일윈드 씨에스에스)를 사용했고,

Backend(백엔드)는 Python(파이썬) FastAPI(패스트에이피아이), Selenium(셀레니움), BeautifulSoup(뷰티풀숲), Pandas(판다스)를 사용했습니다.

AI는 Ollama(올라마)의 Llama 3.1(라마 쓰리 포인트 원)과 ChromaDB(크로마디비) 벡터 데이터베이스를 사용했고,

통신은 REST API와 SSE(Server-Sent Events, 서버 센트 이벤츠)를 사용했습니다.
