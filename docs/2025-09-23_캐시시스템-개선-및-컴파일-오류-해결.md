## 개요

기존 URL 기반 캐시 시스템의 한계를 해결하기 위해 통계명 기반 캐시 시스템으로 개선하고, 프론트엔드 컴파일 오류를 완전히 해결했습니다. 이를 통해 캐시된 데이터의 효율적 활용과 안정적인 애플리케이션 빌드가 가능해졌습니다.

## 주요 변경사항

### 1. 캐시 시스템 구조적 개선

#### 문제점
- 기존 URL 기반 캐시 매칭 방식이 불안정함
- URL의 미세한 차이로 인해 캐시 히트율 저조
- 7개의 캐시된 통계 파일이 있음에도 불구하고 재수집 발생
- 한국어 통계명 정규화 처리 부족

#### 해결방법

**1단계: 통계명 기반 캐시 조회 시스템 구현**
```python
# backend/app/services/data_storage.py
def get_cached_data_by_name(self, stat_name: str) -> Optional[Dict]:
    """통계명으로 캐시된 데이터 조회"""
    normalized_name = self._normalize_stat_name(stat_name)

    for filename in os.listdir(self.statistics_dir):
        if filename.endswith('_stats.json'):
            cached_data = self._load_json_file(
                os.path.join(self.statistics_dir, filename)
            )
            if cached_data and 'stat_name' in cached_data:
                cached_normalized = self._normalize_stat_name(cached_data['stat_name'])
                if cached_normalized == normalized_name:
                    return cached_data
    return None

def _normalize_stat_name(self, stat_name: str) -> str:
    """한국어 통계명 정규화"""
    if not stat_name:
        return ""

    # 공백 정규화 및 특수문자 제거
    normalized = re.sub(r'\s+', '', stat_name.strip())
    normalized = re.sub(r'[^\w가-힣]', '', normalized)
    return normalized.lower()
```

**2단계: API 엔드포인트 캐시 우선 조회 적용**
```python
# backend/app/api/stats.py
@router.post("/basic-statistics-analysis")
async def analyze_basic_statistics(request: BasicStatisticsRequest):

    # ... 기존 분석 로직
```

#### 결과
- ✅ 통계명 기반으로 정확한 캐시 조회 가능
- ✅ 한국어 텍스트 정규화로 매칭 정확도 향상
- ✅ 기존 7개 캐시 파일 모두 활용 가능
- ✅ 불필요한 재수집 방지로 성능 대폭 향상
