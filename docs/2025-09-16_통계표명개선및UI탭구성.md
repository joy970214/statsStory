# 통계표명 개선 및 UI 탭 구성 업데이트

## 개요
통계 데이터 수집 및 표시 시스템의 테이블 명명 규칙 개선과 사용자 인터페이스 탭 구성을 개선하여 더 나은 사용자 경험을 제공하는 업데이트입니다.

## 문제점과 해결방법

### 1. 통계표명 표시 문제

#### 문제점
- 통계표명이 빈 값이거나 "기본 통계표", "테이블1", "테이블2" 같은 의미 없는 이름으로 표시됨
- 사용자가 어떤 통계표인지 구분하기 어려움
- 백엔드와 프론트엔드 간 테이블명 처리 로직 불일치

#### 해결방법
**백엔드 (stats.py)**: 5개 API 엔드포인트에 통계표명 개선 로직 추가
```python
# 테이블명이 없거나 기본값인 경우 개선된 이름 사용
if not table_name or table_name in ['', '기본 통계표']:
    if metadata and metadata.keywords:
        table_name = f"{metadata.keywords[0]} 통계표 {table_counter}"
    else:
        table_name = f"통계표 {table_counter}"
    table_counter += 1
elif table_name.startswith('테이블') and table_name[2:].isdigit():
    if metadata and metadata.keywords:
        table_name = f"{metadata.keywords[0]} {table_name}"
    else:
        table_name = f"수집된 {table_name}"
```

**프론트엔드 (EnhancedBasicStatisticsViewer.tsx)**: 동일한 로직으로 일관성 확보
```typescript
// Enhanced table name processing (matching backend logic)
if (!tableName || tableName === '' || tableName === '기본 통계표') {
  const keywords = analysisData.metadata?.keywords;
  if (keywords && keywords.length > 0) {
    tableName = `${keywords[0]} 통계표 ${tableCounter}`;
  } else {
    tableName = `통계표 ${tableCounter}`;
  }
  tableCounter++;
}
```

### 2. 데이터 수집 실패 시 처리 문제

#### 문제점
- 통계표 수집 실패 시 해당 테이블이 완전히 누락됨
- 사용자가 수집 실패 여부를 알 수 없음
- 부분적 수집 실패도 전체 실패로 처리됨

#### 해결방법
**크롤러 (optimized_molit_crawler.py)**: 실패 시 더미 데이터 생성
```python
if table_data and len(table_data) > 0:
    data_by_table[table_name] = table_data
else:
    # 수집 실패 시 더미 데이터 생성
    dummy_data = [StatData(
        year=datetime.now().year,
        data={"수집상태": "데이터 수집 실패", "테이블명": table_name},
        table_name=table_name,
        collection_status="failed"
    )]
    data_by_table[table_name] = dummy_data
```

### 3. 데이터 저장/로드 시 메타데이터 손실 문제

#### 문제점
- StatData 모델의 추가 필드들(table_name, collection_status 등)이 저장/로드 시 누락됨
- 데이터 품질 정보 손실
- 수집 상태 정보 미보존

#### 해결방법
**데이터 저장 서비스 (data_storage.py)**: 전체 StatData 필드 보존
```python
# 추가 필드들도 저장
if hasattr(stat, 'table_name') and stat.table_name:
    stat_dict['table_name'] = stat.table_name
if hasattr(stat, 'collection_status') and stat.collection_status:
    stat_dict['collection_status'] = stat.collection_status
# ... 기타 필드들
```

### 4. 사용자 인터페이스 개선 문제

#### 문제점
- 모든 분석 내용이 한 화면에 표시되어 복잡함
- 특정 분석 영역을 찾기 어려움
- 긴 스크롤로 인한 사용성 저하

#### 해결방법
**프론트엔드 UI**: 탭 기반 인터페이스 구현
```typescript
const [activeTab, setActiveTab] = useState<'overview' | 'tables' | 'distribution' | 'inspection'>('overview');

// 탭 네비게이션 구현
{activeTab === 'overview' && <OverviewSection />}
{activeTab === 'tables' && <TablesAnalysisSection />}
{activeTab === 'distribution' && <DistributionSection />}
{activeTab === 'inspection' && <InspectionSection />}
```

### 5. 통계표 선택 및 분석 기능 부족

#### 문제점
- 개별 통계표 상세 분석 기능 부족
- 통계표별 데이터 비교 어려움
- 테이블 선택 UI 없음

#### 해결방법
**통계표별 분석 탭**: 개별 테이블 분석 기능 추가
```typescript
// 통계표 선택 버튼
{Object.keys(rawDataByTable).map((tableName, index) => (
  <button
    key={index}
    onClick={() => setSelectedTableName(tableName)}
    className={selectedTableName === tableName ? 'selected' : 'default'}
  >
    {tableName}
  </button>
))}

// 선택된 테이블 상세 정보 표시
{selectedTableName && (
  <TableDetailAnalysis tableData={rawDataByTable[selectedTableName]} />
)}
```
