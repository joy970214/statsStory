# 통계명 동적 저장 시스템 및 취소 기능 개선

**날짜:** 2025-09-23
**작업자:** Claude Code
**카테고리:** 기능 개선, UX 향상

## 개요

기존 정적으로 하드코딩된 통계명 매핑을 동적 저장 시스템으로 개선하고, 분석 취소 기능의 사용자 경험을 향상시켰습니다.

## 주요 변경사항

### 1. 통계명 동적 저장 시스템 구현

#### 문제점
- `optimized_molit_crawler.py`에서 통계명이 정적으로 하드코딩됨
- 새로운 통계 ID 추가 시 수동으로 코드 수정 필요
- 확장성 부족

#### 해결방법
**1단계: 동적 저장소 서비스 생성**
```python
# backend/app/services/stat_name_storage.py
class StatNameStorage:
    def store_stat_name(self, hrsid: str, stat_name: str)
    def get_stat_name(self, hrsid: str) -> Optional[str]
    def has_stat_name(self, hrsid: str) -> bool
```

**2단계: 크롤러 수정**
- 정적 `stat_name_map` → 동적 저장소 조회로 변경
- `_extract_stat_name_from_url()` 함수에서 동적 저장소 사용

**3단계: API 연동**
- 분석 시작 시 통계명 자동 저장하는 헬퍼 함수 추가
- 모든 분석 API에 통계명 저장 로직 적용

#### 결과
- ✅ 최신통계목록에서 "분석하기" 클릭 시 통계명 자동 저장
- ✅ 이후 동일한 통계 접근 시 저장된 이름 사용
- ✅ JSON 파일 기반 영구 저장
- ✅ 기존 하드코딩된 값들도 초기값으로 보존

### 2. 분석 취소 기능 개선

#### 문제점
- 취소 API에서 `ProgressTracker.set_task_status` 메서드 호출 오류
- 취소 중 사용자 피드백 부족
- 취소 진행 중 다른 버튼 클릭 가능 (혼란 야기)

#### 해결방법

**1단계: API 오류 수정**
```python
# 변경 전 (오류)
progress_tracker.set_task_status(task_id, {...})  # 존재하지 않는 메서드

# 변경 후 (정상)
progress_tracker.update_progress(
    task_id=task_id,
    stage="cancelled",
    progress=100,
    message="사용자에 의해 취소됨"
)
```

**2단계: 프론트엔드 UX 개선**
- `isCancelling` 상태 변수 추가
- 취소 버튼: "취소" → "취소 중..." 텍스트 변경
- 취소 중 모든 버튼 비활성화 및 스타일 변경

**3단계: StatCard 컴포넌트 개선**
- `disabled` prop 추가
- 취소 중 "분석하기" 버튼 → "취소 중..." 표시

#### 결과
- ✅ 취소 API 오류 해결
- ✅ 취소 중 명확한 시각적 피드백 제공
- ✅ 취소 진행 중 실수로 다른 작업 시작 방지
- ✅ 사용자 경험 대폭 향상

## 기술적 세부사항

### 통계명 저장 흐름
1. 사용자가 최신통계목록에서 "분석하기" 클릭
2. 프론트엔드에서 `stat_name`과 `stat_url` 전송
3. 백엔드에서 URL에서 `hRsId` 추출
4. `StatNameStorage`에 `hRsId: stat_name` 매핑 저장
5. 이후 동일한 `hRsId` 접근 시 저장된 이름 사용

### 취소 기능 개선 세부사항
- 취소 API 호출 시 `update_progress()` 메서드 사용
- `active_tasks`에 `cancelled` 플래그 설정
- 프론트엔드에서 취소 상태 관리 및 UI 반영
