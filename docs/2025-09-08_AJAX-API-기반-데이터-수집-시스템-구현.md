# AJAX API 기반 통계 데이터 수집 시스템 구현 (2025-09-08)

## 문제 상황

### 1. 기존 크롤링 방식의 한계
**문제**: 기존 IBSheet JavaScript 기반 크롤링이 원본 사이트와 다른 형태의 데이터 제공
**원인**:
- Selenium을 통한 HTML 파싱 방식으로 IBSheet 동적 데이터 불완전 수집
- 실제 사이트는 AJAX API 호출을 통해 구조화된 데이터 제공
- 데이터 검사 시 원본 사이트와 다른 형태로 표시

### 2. 데이터 수집량 제한 문제
**문제**: 크롤링 시 임의로 데이터 양을 제한하여 부분적 데이터만 수집
**원인**:
- 50행, 20행, 3개 테이블 등 하드코딩된 제한
- 성능 최적화 목적이었으나 데이터 완성도 저하
- 전체 통계 분석에 필요한 완전한 데이터셋 확보 실패

### 3. 실제 사이트 동작 방식과 불일치
**문제**: 실제 통계포털의 데이터 로딩 방식과 크롤링 방식이 상이
**원인**:
- 실제 사이트: `columns.do` + `data.do` API 조합으로 데이터 제공
- 기존 방식: JavaScript 실행 후 HTML 파싱으로 데이터 추출
- 컬럼 헤더 구조 정보와 실제 데이터 분리 처리 미구현

## 해결 방법

### 1. AJAX API 패턴 분석 및 직접 호출 시스템

**실제 네트워크 요청 분석:**
```javascript
// 1. 컬럼 구조 정보
GET https://stat.molit.go.kr/portal/stat/columns.do?formId=5374&styleNum=1

// 2. 실제 데이터
GET https://stat.molit.go.kr/portal/stat/data.do?formId=5374&styleNum=1&apprYn=Y&startDate=202507&endDate=202507
```

**구현된 API 직접 호출:**
```python
async def _extract_data_via_api_direct(self, form_id: str, start_date: str, end_date: str):
    """직접 API 호출 (브라우저 세션 없이)"""
    async with aiohttp.ClientSession() as session:
        # 1. 컬럼 정보 요청
        columns_url = f"https://stat.molit.go.kr/portal/stat/columns.do?formId={form_id}&styleNum=1"
        columns_data = await session.get(columns_url).json()
        
        # 2. 데이터 요청
        data_url = f"https://stat.molit.go.kr/portal/stat/data.do?formId={form_id}&styleNum=1&apprYn=Y&startDate={start_date}&endDate={end_date}"
        data_response = await session.get(data_url).json()
        
        # 3. 구조화된 데이터 반환
        return self._structure_api_data(columns_data['data'], data_response['data'])
```

### 2. IBSheet 스타일 데이터 모델 구현

**원본 사이트와 동일한 구조:**
```python
class TableColumn(BaseModel):
    """테이블 컬럼 정보"""
    id: str
    name: str
    data_type: str = "text"  # text, number, date

class TableRow(BaseModel):
    """테이블 행 데이터"""
    row_id: str
    cells: Dict[str, Any]  # column_id -> value

class StatTable(BaseModel):
    """IBSheet 스타일 통계 테이블"""
    table_name: str
    form_id: str
    period: str
    columns: List[TableColumn]
    rows: List[TableRow]
    total_rows: int
    collection_method: str = "api"

class InspectionResult(BaseModel):
    """데이터 검사 결과"""
    tables: List[StatTable]
    total_data_points: int
    collection_success: bool
```

### 3. 향상된 데이터 검사 API 구현

**새로운 엔드포인트:**
```python
@router.get("/inspect-enhanced/{stat_name}", response_model=InspectionResult)
async def inspect_enhanced_data(stat_name: str):
    """IBSheet 스타일 데이터 검사 - API 기반"""
    
    # 1. 기존 저장된 데이터에서 FormId 추출
    base_form_id = extract_form_id_from_url(stat_url)
    
    # 2. 통계표별 데이터 수집
    stat_tables_with_conditions = await crawler._get_stat_tables_with_conditions(stat_url)
    
    # 3. 각 통계표별 API 직접 호출
    for table_info in stat_tables_with_conditions:  # 모든 테이블
        form_id = table_info.get('form_id', base_form_id)
        api_data = await crawler._extract_data_via_api_direct(form_id, current_month, current_month)
        
        # 4. StatTable 객체 생성
        stat_table = StatTable(
            table_name=table_info['name'],
            form_id=form_id,
            columns=[TableColumn(id=col_id, name=col_name) for col_id, col_name in api_data['headers'].items()],
            rows=[TableRow(row_id=f"row_{i}", cells=row_data) for i, row_data in enumerate(api_data['rows'])],
            collection_method="api"
        )
```

### 4. 데이터 수집 제한 완전 제거

**Before (제한된 수집):**
```python
# 행 수 제한
```

### 5. API 우선 + Selenium Fallback 하이브리드 시스템

**처리 흐름:**
```python

except Exception as e:
```

## 성과 및 개선점

### 1. 데이터 품질 대폭 향상
- ✅ **원본 사이트와 동일한 구조**: 컬럼 헤더 + 행 데이터 완벽 분리
- ✅ **전체 데이터 수집**: 10행, 50행 제한 없이 모든 데이터 수집
- ✅ **구조화된 데이터**: JSON 형태의 명확한 데이터 구조

### 2. 성능 및 안정성 개선
- ✅ **API 직접 호출**: Selenium 오버헤드 없는 빠른 데이터 수집
- ✅ **하이브리드 시스템**: API 실패 시 Selenium fallback으로 안정성 확보
- ✅ **실시간 데이터**: 캐시 없이 최신 통계 데이터 즉시 수집

### 3. 사용자 경험 개선
- ✅ **원본 사이트와 동일한 UX**: IBSheet 스타일 테이블 표시
- ✅ **완전한 데이터 검사**: 모든 통계표, 모든 행 데이터 표시
- ✅ **새로운 API 엔드포인트**: `/api/inspect-enhanced/{stat_name}`

### 4. 확장성 확보
- ✅ **FormId 기반 시스템**: 모든 통계표에 대해 일관된 API 호출
- ✅ **다양한 날짜 형식 지원**: YYYY, YYYY-MM, YYYYMM 자동 감지
- ✅ **모듈화된 구조**: 새로운 통계 추가 시 최소한의 코드 수정
