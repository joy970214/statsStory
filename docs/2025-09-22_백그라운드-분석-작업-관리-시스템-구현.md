# 백그라운드 분석 작업 관리 시스템 구현

**날짜**: 2025-09-22
**작성자**: Claude Code
**분류**: 시스템 개선

## 📋 문제 상황

### 1. 백그라운드 작업 상태 손실
- 사용자가 분석 진행 중 로고를 클릭하여 메인 화면으로 이동하면 백엔드에서는 계속 데이터 수집이 진행됨
- 프론트엔드에서는 진행 상황을 알 수 없어 사용자 경험 저하
- 브라우저 새로고침 시 진행 중인 작업 정보 완전 손실

### 2. EventSource 연결 정리 부족
- 컴포넌트 언마운트 시에만 EventSource 정리
- 브라우저 종료, 페이지 새로고침 시 EventSource 연결이 제대로 정리되지 않음
- 메모리 누수 및 불필요한 네트워크 연결 유지

### 3. 작업 취소 기능 부재
- 진행 중인 분석을 중단할 수 있는 방법 없음
- 잘못된 분석 시작 시 다시 시작하려면 완료까지 대기해야 함

### 4. 동시 분석 충돌 문제
- 여러 통계에 대해 동시에 분석 시작 가능
- 서버 리소스 과부하 및 결과 혼재 가능성

### 3. 백엔드 작업 취소 API

**구현 위치**: `backend/app/api/stats.py`

```python
@router.delete("/analysis/cancel/{task_id}")
async def cancel_analysis(task_id: str):
    """분석 작업 취소"""
    try:
        status = progress_tracker.get_task_status(task_id)

        if not status:
            # 작업이 없어도 성공으로 처리 (관대한 처리)
            return {"message": "작업이 이미 완료되었거나 존재하지 않습니다", "task_id": task_id}

        if status.get("completed", False):
            return {"message": "이미 완료된 작업입니다", "task_id": task_id}

        # 작업 취소 - Redis에서 상태 업데이트
        progress_tracker.set_task_status(task_id, {
            "task_id": task_id,
            "completed": True,
            "progress": 100,
            "stage": "cancelled",
            "message": "사용자에 의해 취소됨",
            "cancelled": True
        })

        # 메모리에서 결과 제거
        if task_id in task_results:
            del task_results[task_id]

        return {"message": "작업이 취소되었습니다", "task_id": task_id}

    except Exception as e:
        raise HTTPException(status_code=500, detail=f"작업 취소 오류: {str(e)}")
```

**특징**:
- 404 에러 대신 관대한 처리로 사용자 경험 개선
- Redis 상태 업데이트로 진행률 추적기에 취소 상태 반영
- 상세한 로깅으로 디버깅 용이

### 4. EventSource 연결 정리 강화

**구현 위치**: `frontend/src/components/RealTimeProgressViewer.tsx`

```typescript
useEffect(() => {
  let sse: EventSource | null = null;

  // 브라우저 종료 시 EventSource 정리
  const handleBeforeUnload = () => {
    if (sse) {
      sse.close();
    }
  };

  // 페이지 가시성 변경 시 처리
  const handleVisibilityChange = () => {
    if (document.hidden) {
      console.log('페이지가 숨겨짐 - SSE 연결 상태 확인');
    } else {
      console.log('페이지가 다시 보임 - SSE 연결 상태 확인');
    }
  };

  window.addEventListener('beforeunload', handleBeforeUnload);
  document.addEventListener('visibilitychange', handleVisibilityChange);

  // SSE 연결 설정
  sse = statsAPI.subscribeToProgress(taskId, onProgress);

  return () => {
    // 컴포넌트 정리
    if (sse) {
      sse.close();
    }
    window.removeEventListener('beforeunload', handleBeforeUnload);
    document.removeEventListener('visibilitychange', handleVisibilityChange);
  };
}, [taskId]);
```

**개선사항**:
- `beforeunload` 이벤트로 브라우저 종료 시 EventSource 정리
- `visibilitychange` 이벤트로 탭 전환 모니터링
- 모든 이벤트 리스너 정리로 메모리 누수 방지

### 5. 분석 충돌 방지 시스템

**구현 위치**: `frontend/src/App.tsx`

```typescript
const handleStatSelect = async (stat: StatItem) => {
  // 진행 중인 작업이 있으면 취소 확인
  if (ongoingTask && currentTaskId) {

      return; // 사용자가 취소하면 아무것도 하지 않음

      console.error('기존 작업 취소 실패:', cancelError);
    }
  }

  // 새 분석 시작
  // ...
};
```

**효과**:
- 동시 분석 방지로 서버 리소스 보호
- 사용자 의도 확인으로 실수 방지
- 자동 기존 작업 취소로 편의성 제공

## 🔍 기술 세부사항

### 상태 관리 구조
```typescript
interface OngoingTask {
  taskId: string;        // 백엔드 작업 ID
  statName: string;      // 분석 중인 통계명
  startTime: string;     // 시작 시간 (ISO 형식)
  statInfo: StatItem;    // 통계 상세 정보 (진행 상황 복귀용)
}
```

### API 엔드포인트
- `DELETE /api/analysis/cancel/{task_id}`: 작업 취소
- `GET /api/analysis/status/{task_id}`: 작업 상태 조회
- `GET /api/analysis/progress/{task_id}`: SSE 진행률 스트림

### 2. 작업 히스토리
- 완료/취소된 작업 내역 저장
- 재실행 기능 제공

### 3. 알림 시스템 고도화
- 브라우저 알림 API 활용
- 작업 완료 시 자동 알림

### 4. 실시간 동기화
- WebSocket 기반 실시간 상태 동기화
- 다중 탭에서 동일한 작업 상태 공유

## 📝 주의사항

1. **localStorage 의존성**: 사용자가 브라우저 데이터를 삭제하면 진행 중인 작업 정보 손실
2. **백엔드 재시작**: 백엔드 서버 재시작 시 Redis 데이터 초기화로 작업 상태 불일치 가능
3. **브라우저 호환성**: EventSource는 IE에서 지원하지 않음 (현재 프로젝트는 모던 브라우저 대상)
