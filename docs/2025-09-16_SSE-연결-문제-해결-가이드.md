# SSE(Server-Sent Events) 연결 문제 해결 가이드

## 📋 문제 개요
- **발생일**: 2025-09-16
- **증상**: 기본통계현황분석 시 프론트엔드에서 "SSE 연결을 기다리고 있습니다. 네트워크 상태를 확인해주세요." 메시지가 지속적으로 표시
- **영향**: 실시간 진행률 표시 기능 완전 중단
- **원인**: 백엔드 SSE 구현 및 프론트엔드 EventSource 연결 문제

---

## 🔍 발견된 문제들

### 1. 백엔드 변수 정의 오류

#### 🚨 문제
**파일**: `backend/app/api/stats.py:446, 450`
```python
# 정의되지 않은 progress_callback 변수 사용
progress_callback.update("완료", 100, "분석이 완료되었습니다")
progress_callback.update("오류", 100, f"분석 중 오류 발생: {str(e)}")
```

**에러 로그**:
```
NameError: name 'progress_callback' is not defined
```

#### ✅ 해결방법
```python
# 올바른 ProgressCallback 인스턴스 생성
progress_callback = ProgressCallback(task_id)

# 예외 처리에서는 progress_tracker 직접 사용
progress_tracker.update_progress(task_id, "오류", 100, f"분석 중 오류 발생: {str(e)}")
```

### 2. SSE 응답 헤더 부족

#### 🚨 문제
**파일**: `backend/app/services/progress_service.py`
```python
# 불완전한 SSE 헤더 설정
headers={
    "Cache-Control": "no-cache",
    "Connection": "keep-alive",
    "Access-Control-Allow-Origin": "*",
    "Access-Control-Allow-Headers": "Cache-Control"
}
```

**증상**: 브라우저가 SSE 스트림을 올바르게 인식하지 못함

#### ✅ 해결방법
```python
# 완전한 SSE 헤더 설정
headers={
    "Cache-Control": "no-cache, no-store, must-revalidate",
    "Pragma": "no-cache",
    "Expires": "0",
    "Connection": "keep-alive",
    "Access-Control-Allow-Origin": "*",
    "Access-Control-Allow-Headers": "*",
    "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
    "X-Accel-Buffering": "no",
    "Content-Type": "text/event-stream; charset=utf-8"
}
```

### 3. SSE 메시지 형식 문제

#### 🚨 문제
**증상**: 백엔드에서 메시지를 전송하지만 프론트엔드가 수신하지 못함
```python
# 잠재적인 JSON 직렬화 문제
yield f"data: {json.dumps(asdict(update))}\n\n"
```

#### ✅ 해결방법
```python
# 안전한 JSON 직렬화와 명시적 형식
async def event_generator():
    try:
        # 연결 확인 메시지
        connection_data = json.dumps({
            'type': 'connection', 
            'message': 'Connected', 
            'timestamp': datetime.now().isoformat()
        })
        yield f"data: {connection_data}\n\n"
        
        # 진행률 업데이트
        update_data = json.dumps(asdict(update))
        yield f"data: {update_data}\n\n"
        
    except Exception as e:
        print(f"[SSE] event_generator 오류: {e}")
        # 에러 처리 로직
```

### 4. 프론트엔드 프록시 연결 문제

#### 🚨 문제
**증상**: EventSource readyState가 0 (CONNECTING)에서 변화하지 않음
```javascript
// 프록시를 통한 연결 시도
const sseUrl = `/api/analysis/progress/${taskId}`;
```

**로그**:
```
[API] 5초 후 SSE 상태 체크:
  - readyState: 0  // CONNECTING 상태에서 변화 없음
```

#### ✅ 해결방법
```javascript
// 직접 백엔드 연결로 우회
const sseUrl = `http://localhost:8001/api/analysis/progress/${taskId}`;
```

### 5. 백그라운드 정리 작업 미실행

#### 🚨 문제
**파일**: `backend/main.py`
```python
# 백그라운드 정리 작업이 시작되지 않음
# progress_service의 cleanup 함수가 호출되지 않아 메모리 누수 가능성
```

#### ✅ 해결방법
```python
from app.services.progress_service import start_background_cleanup

@app.on_event("startup")
async def startup_event():
    """앱 시작 시 백그라운드 정리 작업 시작"""
    start_background_cleanup()
    print("백그라운드 정리 작업 시작됨")
```

### 6. OPTIONS 요청 처리 부족

#### 🚨 문제
**증상**: CORS preflight 요청 실패로 SSE 연결 차단
```
브라우저에서 OPTIONS 요청이 실패하여 실제 GET 요청이 차단됨
```

#### ✅ 해결방법
```python
@router.options("/analysis/progress/{task_id}")
async def options_analysis_progress(task_id: str):
    """SSE OPTIONS 요청 처리"""
    from fastapi.responses import Response
    response = Response()
    response.headers["Access-Control-Allow-Origin"] = "*"
    response.headers["Access-Control-Allow-Methods"] = "GET, OPTIONS"
    response.headers["Access-Control-Allow-Headers"] = "*"
    return response
```

## 🔧 기술적 개선사항

### 1. SSE 연결 안정성
- **CORS 설정 완료**: preflight 요청 처리로 브라우저 호환성 확보
- **헤더 최적화**: 캐시 비활성화 및 스트리밍 최적화
- **에러 처리 강화**: 연결 실패 시 자동 재시도 및 상세 로깅

### 2. 메시지 전송 신뢰성
- **JSON 직렬화 안정화**: 각 메시지별 개별 직렬화로 오류 방지
- **하트비트 시스템**: 30초마다 연결 상태 확인
- **타임아웃 처리**: 무한 대기 방지

### 3. 디버깅 시스템
- **단계별 로깅**: 연결 과정의 모든 단계를 추적 가능
- **상태 모니터링**: EventSource readyState 실시간 확인
- **네트워크 디버깅**: 요청/응답 헤더 완전 로깅

### 4. 사용자 경험 개선
- **연결 상태 표시**: 실시간 연결 상태 시각적 피드백
- **진행률 정확도**: 예상 남은 시간 및 단계별 진행률
- **오류 처리**: 사용자 친화적 오류 메시지

## 📚 참고 사항

### 개발 환경 요구사항
- **백엔드**: Python 3.8+, FastAPI, uvicorn
- **프론트엔드**: React 19+, TypeScript 4.9+
- **브라우저**: Chrome 90+, Firefox 88+, Safari 14+

### 모니터링 포인트
- EventSource readyState 변화 (0 → 1 → 메시지 수신)
- 백엔드 SSE 로그 출력 여부
- 네트워크 탭에서 `text/event-stream` 응답 확인
- CORS preflight 요청 성공 여부

### 알려진 제한사항
- **프록시 의존성**: React 개발 서버 프록시를 우회한 직접 연결 사용
- **CORS 설정**: 프로덕션 환경에서는 특정 도메인으로 제한 필요
- **브라우저 지원**: Internet Explorer는 SSE 미지원
