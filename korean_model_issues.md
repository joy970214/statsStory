# 한국어 모델로 변경 시 예상되는 문제점

## 1. 모델 크기 및 성능 문제
- **메모리 사용량 증가**: 현재 8B 모델 → 한국어 모델(3B~32B)로 변경 시 메모리 부족 가능
- **속도 저하**: 이미 CPU만 사용 중인데, 더 큰 모델은 더 느려질 수 있음
- **GPU 없이 실행**: 현재 `size_vram: 0`으로 CPU만 사용 중 → 한국어 모델도 느릴 것

## 2. 모델 다운로드 문제
- **대용량 다운로드**: 한국어 모델은 보통 4GB ~ 20GB 크기
- **인터넷 연결 필요**: Ollama로 모델을 pull 해야 함
- **저장공간**: 추가 모델을 위한 디스크 공간 필요

## 3. 호환성 문제
- **프롬프트 형식**: 각 모델마다 선호하는 프롬프트 형식이 다를 수 있음
- **컨텍스트 길이**: 현재 8192 토큰 → 일부 한국어 모델은 더 작을 수 있음
- **응답 형식**: 파싱 로직이 맞지 않을 수 있음 (현재 `_parse_descriptive_insights` 함수)

## 4. 품질 문제
- **전문 용어**: 통계 전문 용어를 잘 이해하지 못할 수 있음
- **구조화된 출력**: 현재 프롬프트의 "1-1, 1-2, 3-1..." 형식을 제대로 따르지 않을 수 있음
- **일관성**: 응답 품질이 일정하지 않을 수 있음

## 5. 유지보수 문제
- **업데이트**: 한국어 모델은 업데이트가 덜 빈번함
- **커뮤니티 지원**: 영어 모델보다 문서와 커뮤니티 지원이 적음
- **버그 수정**: 한국어 특화 버그가 있을 수 있음

## 해결 방안

### Option A: 하이브리드 접근 (추천)
1. **프롬프트를 영어로 작성**, 응답만 한국어로 요청
   ```python
   prompt = """You are a statistics analyst. Analyze this Korean data and respond in Korean.

   Data: [한국어 데이터]

   Response format:
   1-1. Analysis Title (in Korean)
   ...
   """
   ```
   - 장점: 현재 모델 유지, 한국어 응답 가능
   - 단점: 한국어 데이터 이해도는 여전히 낮을 수 있음

### Option B: 작은 한국어 모델 테스트
1. **exaone3.5:2.4b** 같은 작은 모델로 테스트
   ```bash
   ollama pull exaone3.5:2.4b
   ```
2. 코드 수정:
   ```python
   model: str = "exaone3.5:2.4b"
   ```
   - 장점: 메모리 사용량 적음, 한국어 이해도 높음
   - 단점: 분석 품질이 떨어질 수 있음

### Option C: 현재 모델 최적화 (가장 안전)
1. **프롬프트 크기 줄이기**
   - 데이터 샘플 수: 50개 → 20개
   - 통계표 설명 간소화
2. **타임아웃 늘리기**
   - 600초 → 900초 (15분)
3. **num_predict 조정**
   - 500 → 300 (더 짧은 응답)

## 추천 순서
1. **먼저 현재 문제 해결** (Option C)
   - 왜 현재 모델이 실패하는지 확인
   - 프롬프트 최적화
2. **작은 한국어 모델 테스트** (Option B)
   - 성공하면 계속 사용
   - 실패하면 Option A
3. **하이브리드 접근** (Option A)
   - 마지막 수단
