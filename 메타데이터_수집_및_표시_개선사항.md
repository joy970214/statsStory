# 메타데이터 수집 및 표시 개선사항

## 발견된 문제들과 해결방법

### 1. 관련용어 탭 메타데이터 수집 문제

**문제**:
- 통계정보 탭은 th/td HTML 구조로 되어 있어 수집이 잘 되었으나, 관련용어 탭은 th/td 형태가 아니어서 수집이 안되었음
- 기존 로직으로는 테이블 구조가 아닌 텍스트 기반 콘텐츠를 수집할 수 없었음

**해결방법**:
- `backend/app/crawlers/optimized_molit_crawler.py`의 `_extract_terms_details` 메서드를 완전히 개선
- HTML 구조 분석 대신 정규식 패턴 매칭을 사용한 텍스트 기반 수집 로직 구현
- 주요항목, 의미분석, 관련용어 세 섹션을 각각 다른 패턴으로 추출
- "관련용어/" 접두사를 사용하여 카테고리별로 분류 저장

```python
# 개선된 로직 핵심 부분
major_items_pattern = r'주요항목\s*(?:\n|\r\n?)(.*?)(?=\n\s*의미분석|\n\s*관련용어|$)'
meaning_analysis_pattern = r'의미분석\s*(?:\n|\r\n?)(.*?)(?=\n\s*관련용어|$)'
related_terms_pattern = r'관련용어\s*(?:\n|\r\n?)(.*?)$'
```

### 2. 불필요한 데이터 수집 문제

**문제**:
- "리스트항목_1_1" 같은 불필요한 데이터가 수집되었음
- 리스트 아이템 수집 로직이 의도와 다르게 동작

**해결방법**:
- `_extract_terms_details` 메서드에서 리스트 수집 로직 완전 제거
- 텍스트 패턴 기반 수집으로 변경하여 구조화된 콘텐츠만 추출

### 3. 메타데이터 표시 위치 문제

**문제**:
- 수집된 메타데이터가 "통계표별 분석" 탭에 표시되고 있었음
- 사용자는 "전체 요약" 탭에서 메타데이터를 보고 싶어했음

**해결방법**:
- `BasicStatisticsViewer.tsx`에서 메타데이터 섹션 완전 제거
- `EnhancedBasicStatisticsViewer.tsx`의 "전체 요약" 탭에 메타데이터 섹션 추가
- 컴포넌트 식별 과정에서 "기본통계현황분석 결과" 화면이 실제로는 EnhancedBasicStatisticsViewer임을 확인

### 4. 메타데이터 표시 내용 최적화

**문제**:
- 기본 정보(제목, 목적, 주기 등)와 "상세 메타정보" 제목이 불필요하다고 판단됨

**해결방법**:
- 기본 정보 섹션 (제목, 목적, 주기, 담당자, 작성기관, 검색분야, 담당부서, 키워드) 완전 제거
- "상세 메타정보" 제목 제거
- 통계정보상세, 주요항목, 의미분석, 용어정의, 관련용어 섹션만 표시

## 핵심 기술적 개선사항

### 1. 정규식 패턴 매칭 도입
```python
# 한국어 텍스트와 공백 처리를 고려한 정규식
major_items_pattern = r'주요항목\s*(?:\n|\r\n?)(.*?)(?=\n\s*의미분석|\n\s*관련용어|$)'
```

### 2. 텍스트 전처리 강화
```python
# 공백과 줄바꿈 정규화
text = re.sub(r'\s+', ' ', text).strip()
```

### 3. 데이터 구조화
- 카테고리별 접두사 사용: "관련용어/주요항목", "관련용어/의미분석", "관련용어/관련용어"
- 각 항목을 키-값 쌍으로 체계적으로 저장

### 4. 컴포넌트 구조 개선
- 조건부 렌더링을 통한 메타데이터 섹션 표시
- 섹션별 색상 구분으로 가독성 향상
